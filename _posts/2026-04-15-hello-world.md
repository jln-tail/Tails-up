这是本博客的第一篇文章，用来：

1. 说明博客用途；
2. 展示后续写文章会用到的 Markdown 语法；
3. 作为 front matter 模板参考。

## 用途

记录三类内容：

- **Bug 复盘**：某个问题怎么出现、怎么定位、最后怎么修的。
- **踩坑笔记**：某个工具 / 库 / API 的非直觉行为。
- **短笔记**：一条命令、一段配置、一个结论，不展开。

文风尽量直白，不追求完整教程。

## Markdown 语法速览

### 代码块

行内代码用反引号，例如 `git log --oneline -n 5`。

多行代码块带语言标记以便高亮：

```bash
# 查看最近 5 条提交
git log --oneline -n 5
```

```python
def fib(n: int) -> int:
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a
```

### 列表与引用

- 普通列表
- 有序列表用 `1.` `2.` `3.`
- 任务列表：
  - [x] 搭好博客
  - [ ] 写 10 篇笔记

> 引用块用来强调结论或引用他人原话。

### 链接与行内样式

访问 [Jekyll 官网](https://jekyllrb.com/) 了解更多。**加粗**、*斜体*、~~删除线~~ 都支持。

### 表格

| 场景 | 建议 |
| --- | --- |
| 短笔记 | 直接开一篇，不要凑合 |
| 长文 | 先列大纲再写 |

## 新增文章模板

复制以下内容到 `_posts/YYYY-MM-DD-your-slug.md` 即可：

```markdown
---
title: "你的标题"
date: 2026-05-01
tags: [tag1, tag2]
description: "一句话摘要，可省略。"
---

正文从这里开始。
```
